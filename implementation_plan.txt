# Tenant Registration and Lease Assignment

The goal is to allow tenants to fully manage their profile and family details, view vacant properties, and limit their side-menu access until a property is leased to them. Landlords are given complete visibility into tenant profiles (including income) to formalize leases.

## Verification Plan

### Automated Tests
*The repository does not have automated testing set up, so we will rely on manual functional testing.*

### Manual Verification
1. **Database Update:** Run the `ALTER TABLE tenants ADD COLUMN income DECIMAL(10, 2)` command against the local MySQL instance via a node script or shell command.
2. **Tenant Registration & Profile:**
   - Register a new user with the `tenant` role via the `/register` UI.
   - Login, navigate to the **Profile** tab, and fill out details including `income` and add a family member.
   - Verify that **Leases**, **Payments**, and **Complaints** are hidden from the sidebar.
   - Go to **Properties** and verify only vacant properties are displayed without action buttons.
3. **Landlord View & Lease Assignment:**
   - Login as a `landlord` or `admin`.
   - Go to **Tenants** and verify the newly registered tenant appears with their `income` and family members.
   - Go to **Leases** and create a new lease assigning a vacant property to the tenant.
4. **Tenant Menu Visibility Re-check:**
   - Login as the `tenant` again.
   - Verify that **Leases**, **Payments**, and **Complaints** sections now appear in the Sidebar because of the active lease.
